Exquisitus is a Starlight [plugin](https://starlight.astro.build/reference/plugins/).
You add it to an existing Starlight site; it brings its own fonts, palette, and component refinements.

<Steps>

1. Install the theme alongside Starlight.

   <Tabs syncKey="pkg">
   <TabItem label="pnpm">
   	```sh
   	pnpm add starlight-theme-exquisitus
   	```
   </TabItem>
   <TabItem label="npm">
   	```sh
   	npm install starlight-theme-exquisitus
   	```
   </TabItem>
   <TabItem label="Bun">
   	```sh
   	bun add starlight-theme-exquisitus
   	```
   </TabItem>
   </Tabs>

2. Register the plugin in your Starlight configuration.

   ```js title="astro.config.mjs" ins={[2, 8]}
   import starlight from '@astrojs/starlight';
   import starlightThemeExquisitus from 'starlight-theme-exquisitus';
   import { defineConfig } from 'astro/config';

   export default defineConfig({
     integrations: [
       starlight({
         plugins: [starlightThemeExquisitus()],
         title: 'My Docs',
       }),
     ],
   });
   ```

3. Start the dev server and look at the result.

   ```sh
   pnpm dev
   ```

</Steps>

<Aside type="note" title="Search runs in production builds">
	Starlight's search is powered by [Pagefind](https://pagefind.app/). Search indexes the site at
	build time, so the search box is inert under `pnpm dev` and shows a short placeholder instead of
	results. To try search, build and preview:

	```sh
	pnpm build && pnpm preview
	```
</Aside>

<Aside type="tip" title="Bring your own brand">
	Every color is a CSS custom property.
	Override `--sl-color-accent` and friends in your own
	stylesheet to retune the palette without forking the theme.
</Aside>

## What the plugin does

Registering the plugin makes three changes to your site:

1. It self-hosts three typefaces: Atkinson Hyperlegible Next, Spectral, and JetBrains Mono get added via `@fontsource`, so nothing is fetched from a third-party origin.
2. It replaces the Starlight color system with an OKLCH palette for light and dark modes.
   Colors are layered so your own `customCss` still wins.
3. It refines the core components—header, sidebars, asides, cards, code blocks, file trees, and the hero—without changing Starlight's markup or behavior.

<Aside type="note">
	Exquisitus shouldn't override a component you have already set yourself.
	If there is a conflict, it should	step aside and log a warning so your configuration stays in charge.
</Aside>
