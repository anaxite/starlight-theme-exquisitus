Starlight ships a generous set of documentation components. Exquisitus restyles each one to match the reading column — same hairlines, same palette, same restraint.

## Asides

Each callout takes its hue from the semantic ramp and wears a soft full border instead of a side stripe. The icon and the serif title carry the meaning.

<Aside type="note">A note shares the calm petrol-teal of inline code — it is information, set apart.</Aside>

<Aside type="tip" title="A considered tip">Tips lean violet, distinct from both the warm accent and the cool note.</Aside>

<Aside type="caution">Caution is amber-orange, deliberately separated from the brand's link colour so the two never blur.</Aside>

<Aside type="danger">Danger is unmistakable without being lurid — a measured red that still passes contrast on its own tint.</Aside>

## Tabs

<Tabs>
	<TabItem label="Light">The active tab is marked by the accent underline. Light mode keeps a pure-white page; warmth lives in the ink and the bronze accent.</TabItem>
	<TabItem label="Dark">In dark mode the same accent brightens to honey, and the petrol-teal lifts so both stay legible on near-black.</TabItem>
</Tabs>

## Steps

<Steps>

2. The connecting guide fades into the page rather than ending in a hard stop.
3. Each step is free to hold prose, code, or another component.

</Steps>

## Code blocks

Code keeps Expressive Code's own frames — a titled tab for files, a traffic-light bar for terminals — set on the theme's cool-tinted code surface so blocks read in the information register, distinct from the prose around them.

```ts title="theme.config.ts" {4-6}
import starlightThemeExquisitus from 'starlight-theme-exquisitus';

export const plugins = [
	starlightThemeExquisitus({
		// The theme reads your Expressive Code config and extends it.
	}),
];
```

```sh title="Terminal"
pnpm add starlight-theme-exquisitus
```
